# Reels, Shorts e TikTok — manual de produção

## Quando usar

- Roteirizando qualquer vídeo vertical curto (15-60s)
- Decidindo formato/duração/ritmo
- Diagnosticando por que um Reel/Short performou mal

## Princípio: 90% começa muting

A maioria dos usuários começa o Reel/Short com som desligado. Se sua mensagem **só funciona com áudio**, você perdeu. Texto na tela e visual têm que carregar a mensagem mesmo sem som.

---

## Anatomia: 0-3 segundos é onde você ganha ou perde

### Os primeiros 3 segundos

Algoritmo decide se vai distribuir o vídeo com base em:
- **Hold rate** (quem segura nos primeiros 3s)
- **Replay rate** (quem volta pro início)
- **Like + comment + share** dentro dos primeiros 5s

Se 50% das pessoas dropam nos primeiros 3s, o vídeo morre. Sem chance.

### Como segurar nos primeiros 3s

**Hook visual** (mais importante que o falado):
- Movimento na cena (não plano estático)
- Texto grande, contrastante, na tela
- Close em rosto (humanos prendem em rostos)
- "Padrão quebrado" — algo inesperado nos primeiros frames

**Hook falado** (se o usuário ativou som):
- Frase curta, específica
- Pergunta direta
- Promessa de payoff em segundos

**Hook escrito (text overlay)**:
- 5-7 palavras, máximo
- Posição: terço superior ou central (NÃO no rodapé — UI cobre)
- Cor com contraste alto contra o fundo
- Aparece no frame 1 (não fade in lento)

### Padrões que NÃO seguram

- ❌ Plano estático sem movimento
- ❌ "Fala galera, sejam bem-vindos"
- ❌ Logo abrindo no início (corte. logo no fim, se quiser)
- ❌ Música tocando 2s antes da fala começar
- ❌ Você se apresentando antes de dizer pra que veio
- ❌ Frase longa que demora 4s pra fazer sentido

## Arcos narrativos por duração

### 15 segundos (formato curto, viraliza mais)

```
0-2s   Hook (visual + texto + 1 frase)
2-10s  Desenvolvimento — UM ponto, sem desvio
10-13s Payoff / virada
13-15s CTA visual + falado
```

Exemplo (clínica de estética):
```
[0-2s]   Mulher olhando no espelho. Texto: "Botox cai antes do tempo?"
[2-10s]  "Tem 1 detalhe na aplicação que ninguém te conta:
          se a toxina não chegou no plano muscular certo,
          vai durar metade do tempo."
[10-13s] Imagem técnica simples mostrando profundidade
[13-15s] "Avaliação grátis no link"
```

### 30 segundos (formato sweet spot pra educativo)

```
0-3s   Hook duplo (visual + falado)
3-8s   Setup / contexto
8-22s  Desenvolvimento — pode ter 2-3 sub-pontos
22-27s Payoff
27-30s CTA
```

### 60 segundos (formato pra conteúdo denso)

```
0-3s    Hook
3-10s   Promessa expandida ("vou te mostrar X em Y segundos")
10-50s  Desenvolvimento com 3-5 pontos, cortes a cada 3-4s
50-55s  Payoff / insight central
55-60s  CTA
```

> Vídeo de 60s perde retenção mais rápido. Só use quando o tema realmente pede.

## Ritmo de cortes

| Duração | Cortes        | Tipo de corte                    |
| ------- | ------------- | -------------------------------- |
| 15s     | a cada 1-2s   | Cortes secos, jump cut, zoom in/out |
| 30s     | a cada 2-3s   | Mistura de tipo                  |
| 60s     | a cada 3-5s   | Pode ter planos mais longos no meio |

**Sem cortes** = sem retenção. Mesmo se for talking head, varie ângulo, dê zoom in, jump cut.

## Texto na tela vs. fala (regra crítica)

**Não repita.** Se a fala diz "esse erro custa caro", o texto na tela diz "R$ 2.300 jogados fora". Ou: "muita gente comete".

| Fala diz                          | Texto na tela diz                    |
| --------------------------------- | ------------------------------------ |
| "Esse é o maior erro"             | "ERRO #1"                            |
| "Custa caro"                      | "R$ 2.300 perdidos"                  |
| "Ninguém te conta isso"           | "...e por que ninguém fala?"         |
| "A solução é simples"             | "↓ olha só"                          |

Texto **complementa**, não duplica. Cobre as duas pontas: quem ouve E quem só lê.

## Quando legendar (CC) vs. quando não

**Sempre legenda** quando:
- Vídeo é talking head com fala importante
- Ambiente tem ruído ou som de baixa qualidade
- Conteúdo é técnico/educativo
- Público inclui pessoas com deficiência auditiva (e tem que incluir)

**Não precisa legenda** quando:
- Vídeo é puramente visual (B-roll, demo, transformação)
- Áudio é trend/música (texto na tela substitui)
- Vídeo de 5-10s muito visual

**Estilo de legenda**:
- Sem fundo opaco (parece amador)
- Sem rolar palavra por palavra (cansa)
- 1-3 linhas por vez
- Posição: terço inferior, evitando UI

## Formato técnico

| Plataforma | Aspecto       | Resolução    | Duração max |
| ---------- | ------------- | ------------ | ----------- |
| IG Reels   | 9:16          | 1080x1920    | 90s (até 3min em alguns casos) |
| YT Shorts  | 9:16          | 1080x1920    | 60s         |
| TikTok     | 9:16          | 1080x1920    | 10min (mas sweet spot é 30-60s) |

**Áudio**: 30fps, stereo, -14 LUFS (padrão broadcast). Áudio limpo importa MUITO.

## Trends — usar ou não

### Quando vale
- Trend bate com a marca naturalmente
- Você é dos primeiros a entrar (não o 50.000º)
- Você adapta criativamente (não só copia)

### Quando não vale
- Trend já saturou (depois de 2-3 semanas)
- Marca é premium/B2B/saúde — trends genéricas pegam mal
- Você precisaria forçar pra encaixar
- Trend tem origem polêmica/risco

### Como saber se trend tá quente
- Aba **Discover** do TikTok
- Página de **áudios** do Reels (mostra "em alta")
- Conta de **trend analyst** (Trend Watching, Watchwhattheydo)
- **Rapidez é tudo**: trend dura 5-21 dias

## Estruturas narrativas testadas (escolha uma)

### 1. Problema → solução
0-3s: "Você [tem esse problema]?"
3-25s: "Aqui tá por que acontece e como resolver"
25-30s: CTA

### 2. Lista numerada
0-3s: "3 erros que [público] comete"
3-22s: Erro 1, Erro 2, Erro 3 (cada um 6-7s)
22-27s: O que fazer no lugar
27-30s: CTA

### 3. Mito vs. verdade
0-3s: "Você já ouviu que [mito]?"
3-15s: "Pois é mentira. Aqui o que realmente acontece"
15-25s: Verdade + por quê
25-30s: CTA

### 4. Antes/depois (se a marca permite)
0-3s: Visual do "antes" + texto contextual
3-25s: Processo (não dá detalhe demais)
25-30s: Resultado + CTA

> Cuidado: antes/depois em saúde/estética requer disclaimer. Nem todo CRM/regulamentação permite.

### 5. Confissão / bastidor
0-3s: "Vou contar uma coisa que custou caro pra eu aprender"
3-25s: Narrativa pessoal específica
25-30s: Insight + CTA

### 6. Storytime curto
0-3s: Cena marcante / "essa cliente entrou na clínica chorando..."
3-25s: História com início, meio, virada
25-30s: Lição + CTA

## Erros que matam vídeo curto

- ❌ Hook genérico ("hoje vou falar sobre")
- ❌ Plano estático longo
- ❌ Texto na tela igual à fala
- ❌ CTA múltiplo
- ❌ Música muito alta atrapalhando fala
- ❌ Fala monótona, sem energia
- ❌ Editar no app da plataforma com fonte feia
- ❌ Logo da marca aparecendo a cada 5s (parece desespero)

---

## Checklist antes de publicar

- [ ] Hook visual + falado nos 3 primeiros segundos
- [ ] Texto na tela complementa, não duplica fala
- [ ] Cortes no ritmo certo pra duração
- [ ] Legenda gerada (auto + revisada)
- [ ] CTA único, claro, em texto e fala nos últimos 2-3s
- [ ] Áudio limpo, sem distorção
- [ ] Aspecto 9:16, sem área cortada por UI
- [ ] Capa (cover frame) atrativa — pessoa vai escolher se abre o perfil
- [ ] Legenda do post puxa pra ver até o fim
- [ ] Hashtags apropriadas (3-5 no TikTok, 5-10 no Reels)

## Métricas pra acompanhar

| Métrica            | O que diz                               |
| ------------------ | --------------------------------------- |
| Hold rate (3s)     | Hook funcionou ou não                   |
| Watch time médio   | Conteúdo segurou                        |
| Replay rate        | Conteúdo é replayable (loop)            |
| Like rate          | Ressonância emocional                   |
| Save rate          | Utilidade percebida                     |
| Share rate         | Identificação ("isso é a cara de X")    |
| Comments           | Conversa gerada                         |
| Profile visit rate | CTA "segue" funcionou                   |

Foque em **save + share + watch time**. Like é vanity. Profile visit é fundo.
